# Introduction

## En bref

Canyon est une solution de couverture de code JavaScript. Grâce à une configuration simple de plugin Babel, il est possible de réaliser le téléchargement de la couverture et la génération de rapports en temps réel, résolvant le problème difficile pour les développeurs et les ingénieurs QA de collecter la couverture de code dans les tests end-to-end.

## Diagramme d'architecture

Ce diagramme illustre l'architecture de Canyon et certaines composantes de son écosystème :

1. Le plugin Babel complète l'instrumentation de code dans le processus CI/CD.
2. Le projet est publié dans l'environnement de test. L'utilisateur choisit le test automatisé UI ou le test manuel pour déclencher la sonde de code instrumentée.
3. Les données de couverture générées sont téléchargées vers le serveur Canyon.
4. Le serveur Canyon consomme les données et, en combinaison avec les informations de code source fournies par GitLab, génère un rapport de couverture.
