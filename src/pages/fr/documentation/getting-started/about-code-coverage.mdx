# À propos de la couverture de code

## Qu'est-ce que la couverture de code?

La couverture de code est une mesure utilisée pour indiquer quelles lignes de code ont été exécutées par un ensemble de tests. Elle a quatre dimensions.

- **Couverture des lignes** : Chaque ligne de code a-t-elle été exécutée?
- **Couverture des fonctions** : Chaque fonction a-t-elle été appelée?
- **Couverture des branches** : Chaque branche a-t-elle été exécutée?
- **Couverture des instructions** : Chaque instruction a-t-elle été exécutée?

En termes simples, la couverture de code fournit une mesure visuelle pour montrer quels codes sources ont été exécutés par un ensemble de tests. Cette information peut aider les développeurs de logiciels à déterminer où écrire de nouveaux tests pour s'efforcer d'obtenir une couverture plus élevée.

## Qu'apporte Canyon?

Les outils de couverture de code encouragent les développeurs à écrire des tests et à améliorer la couverture. Au cours de l'écriture des tests, les développeurs peuvent découvrir de nouveaux erreurs ou problèmes de syntaxe dans le code source. Il est très important de résoudre ces problèmes avant de publier l'application.

Canyon se concentre sur la collecte de la couverture des tests end-to-end. La couverture des tests end-to-end est utilisée pour assurer la qualité du code et l'intégrité fonctionnelle de l'application entière sous les opérations des utilisateurs réels. Il vérifie la fonctionnalité de l'application en simulant le comportement des utilisateurs, en identifiant les problèmes d'intégration, en prévenant les erreurs de régression et en aidant à optimiser l'expérience utilisateur. Cette couverture renforce la confiance de l'équipe dans la publication du code et assure la stabilité et la fiabilité de l'application dans son utilisation pratique.
